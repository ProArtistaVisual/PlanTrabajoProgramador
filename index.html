Los 4 pilares del plan (síntesis de los 3 asesores)
🔧
1. Enfoque técnico — romper la burbuja
Tu PHP y SQL son reales y valiosos. El problema no es lo que sabes, sino que está acoplado a un solo contexto. La solución es estandarizar, no empezar de cero.
  • Laravel o Symfony como framework estándar de mercado
  • Patrones de diseño: Repository, Factory, Singleton, SOLID
  • Arquitectura limpia / hexagonal
  • SQL avanzado: CTEs, window functions, índices, transacciones
  • Node.js / NestJS como segundo stack (Mes 5+)
  • PHPUnit: pruebas unitarias e integración
🤖
2. IA como superpoder — no como amenaza
No necesitas construir modelos. Necesitas ser un desarrollador aumentado por IA. La IA no reemplaza a quien sabe; multiplica su capacidad.
  • GitHub Copilot + Cursor como copiloto diario
  • Prompt engineering avanzado para código, tests y docs
  • Integrar APIs de Anthropic / OpenAI en proyectos reales
  • RAG (Retrieval-Augmented Generation) con LangChain
  • LlamaIndex para chatbots sobre datos propios
  • Conocer los límites: saber cuándo el output de la IA es incorrecto
☁️
3. Nube y DevOps — el entorno donde vive el código
Un ingeniero moderno no se limita a escribir código local. Saber desplegar es tan importante como saber programar. Docker primero, cloud después.
  • Docker: contenedores, docker-compose, reproducibilidad
  • CI/CD: GitHub Actions para deploy automático
  • AWS básico: EC2, S3, RDS, Lambda
  • Railway / Render para deploys rápidos y gratuitos
  • AWS Certified Cloud Practitioner (meta Dic 2026)
  • Monitoreo básico: logs, alertas, uptime
🎯
4. Estrategia de carrera — visibilidad y transición
No esperes a que tu ciclo en la empresa termine para saber qué pide el mercado. Construye evidencia pública de tus capacidades ahora.
  • Portafolio propio en GitHub + GitHub Pages
  • Leer código ajeno 20 min/día (repos con ★)
  • LinkedIn optimizado con palabras clave técnicas
  • LeetCode / HackerRank para entrevistas técnicas
  • Contribución a 1 repo open source (al menos 1 PR aceptado)
  • Postulación a 5 posiciones remotas en Dic 2026

Skills de Claude — biblioteca de capacidades reutilizables
Las Skills son archivos SKILL.md que extienden el comportamiento de Claude para tareas específicas. Funciona en Claude Code, Cursor, Codex, Gemini CLI y más. Un repo con 4 reglas llegó a 144k ★ en días.
¿Qué es una Skill de Claude?
Una Skill es un archivo .md con instrucciones, contexto de dominio y guardrails de comportamiento. Claude los carga progresivamente — solo cuando son relevantes — reduciendo el uso de tokens. No son MCP (que conecta herramientas externas) ni tools (funciones invocables): son la capa de workflow y criterio que define cómo actuar una vez conectado.
